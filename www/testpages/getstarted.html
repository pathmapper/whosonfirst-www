Who’s On First allows you to access and utilize place/geo data in a variety of ways. We created a number of tutorials, so you can learn how to do all this yourself.

  • retrieve venues - retrieve venues in New York City’s Flatiron District
  • retrieve neighbourhoods - retrieve San Francisco's neighbourhoods

Within these tutorials, you will have the ability to select from different methods to access the data.

Select the method by clicking on its tab towards the top of its specific page.

We have at most three available options to access the data. These options are:

Option 1: API Query

The Mapzen Places API allows developers (and robots) programmatic access to query and retrieve Who’s On First data via a rest-ish interface.

API methods are dispatched over HTTP with one or more query parameters and data is returned in response as JSON by default but you may also specify CSV or Who’s On First’s own meta formatted responses for certain API methods.

The endpoint for the Mapzen Places API is: https://places.mapzen.com/v1

All errors are returned using HTTP status codes in the 400-599 range.

400 class errors mean the problem was on your end and 500 class errors mean the problem was our fault.

Option 2: HTTP Call

You can also access individual WOF place data by making a http call. For example you can call https://whosonfirst.mapzen.com/data/{PATH}/{ID}.geojson if you know that place’s PATH and ID.

Option 3: Personal Copy

Anyone can host their own copy of WOF data. Our copy of the Who's On First data is stored in big-honking Amazon Web Services (AWS) S3 bucket called whosonfirst.mapzen.com in a sub-directory, or prefix in AWS-speak, called data. If you so desire, you can fetch the entirety of all the contents of this S3 bucket. You could also take bits and pieces by downloading individual GitHub repos.
